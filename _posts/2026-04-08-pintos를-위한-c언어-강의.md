<h2>[강의 요약] C언어의 심장: 포인터와 메모리 관리 완벽 정리</h2>
<p>안녕하세요! 오늘은 프로그래밍의 '근본'이라 불리는 C언어, 그중에서도 많은 학습자가 어려워하지만 반드시 넘어야 할 산인 <strong>메모리와 포인터</strong>에 대해 정리해 보겠습니다. 파이썬처럼 친절한 언어와 C언어는 무엇이 다른지, 왜 우리가 직접 메모리를 만져야 하는지 핵심만 짚어드릴게요.</p>
<hr>
<h3>1. C언어는 왜 '불친절'할까? (vs Python)</h3>
<p>파이썬이 모든 것을 알아서 해주는 '자동 자가용'이라면, C언어는 운전자가 엔진 노즐 하나하나를 조절해야 하는 '수동 레이싱카'와 같습니다.</p>
<ul>
<li><strong>하드웨어 제어:</strong> CPU와 메모리를 직접 컨트롤하기 때문에 성능이 강력합니다.</li>
<li><strong>명시적 타입:</strong> 모든 데이터의 크기를 프로그래머가 직접 결정해야 합니다.</li>
<li><strong>불친절한 에러:</strong> 잘못된 메모리에 접근하면 친절한 설명 대신 <code>Segmentation Fault</code>와 함께 프로그램이 종료됩니다.</li>
</ul>
<h3>2. 변수를 바라보는 새로운 시각: 주소, 값, 그리고 크기</h3>
<p>C언어에서 변수는 단순히 값을 담는 그릇이 아니라, <strong>메모리 상의 특정 위치를 가리키는 별명(Symbol)</strong>입니다.</p>
<ul>
<li><strong>주소(&amp;a):</strong> 메모리 어디에 있는가?</li>
<li><strong>값(a):</strong> 그 위치에 무엇이 들어있는가?</li>
<li><strong>크기(sizeof):</strong> 그 위치부터 몇 바이트를 사용하는가?</li>
</ul>
<blockquote><p><p><strong>💡 중요 팁:</strong> C언어에서 데이터 타입(int, long 등)의 크기는 시스템마다 다를 수 있습니다. 따라서 이식성 있는 코드를 위해 <code>stdint.h</code>의 고정 너비 타입(<code>int32_t</code>, <code>int64_t</code>)을 사용하는 습관이 좋습니다.</p>
</p></blockquote><hr>
<h3>3. 통곡의 벽, '포인터(Pointer)' 제대로 이해하기</h3>
<p>포인터는 어렵지 않습니다. <strong>"주소를 저장하는 변수"</strong>일 뿐입니다.</p>
<ul>
<li><strong>역참조(<code>*</code>):</strong> 포인터가 가리키는 주소로 '직접 찾아가서' 값을 읽거나 쓰는 행위입니다.</li>
<li><strong>포인터 타입의 비밀:</strong> <code>int*</code>와 <code>char*</code>는 주소 값 자체의 크기(64비트 기준 8바이트)는 같지만, <strong>찾아갔을 때 몇 바이트를 읽어야 할지</strong>를 결정합니다.</li>
<li><strong>포인터 연산:</strong> <code>p + 1</code>은 주소 값에 1을 더하는 게 아니라, <strong>가리키는 타입의 크기만큼</strong> 건너뛰는 것을 의미합니다.</li>
</ul>
<hr>
<h3>4. 메모리 관리의 핵심: 힙(Heap)과 동적 할당</h3>
<p>함수가 종료되면 사라지는 지역 변수(Stack)와 달리, 프로그램 실행 중에 원하는 만큼 메모리를 쓰고 싶을 때 <strong>힙(Heap)</strong> 영역을 사용합니다.</p>
<ul>
<li><strong><code>malloc</code>:</strong> 메모리 빌려오기. (반환 타입이 <code>void*</code>이므로 적절한 형변환이 필요합니다.)</li>
<li><strong><code>free</code>:</strong> 빌린 메모리 반납하기. </li>
<li><strong>메모리 누수(Memory Leak):</strong> 빌려놓고 반납(<code>free</code>)하지 않으면 시스템의 메모리가 고갈됩니다. <strong>"malloc과 free는 항상 짝꿍"</strong>임을 잊지 마세요!</li>
</ul>
<hr>
<h3>5. 구조체(Struct)와 메모리 정렬의 마법</h3>
<p>서로 다른 타입을 묶는 구조체에는 숨겨진 원리가 있습니다. 바로 <strong>패딩(Padding)</strong>입니다.</p>
<p>CPU는 메모리 접근 효율을 위해 데이터를 특정 단위로 정렬합니다. 이 과정에서 빈 공간(Padding)이 생겨, 실제 구조체 크기가 멤버들의 합보다 커질 수 있습니다. </p>
<ul>
<li><strong>최적화 팁:</strong> 크기가 큰 타입부터 작은 타입 순서로 멤버를 선언하면 패딩을 줄여 메모리를 아낄 수 있습니다.</li>
</ul>
<hr>
<h3>✅ 오늘 강의의 핵심 '코딩 습관' 5가지</h3>
<ol>
<li><strong>sizeof 생활화:</strong> 타입 크기를 짐작하지 말고 항상 <code>sizeof()</code>로 확인하세요.</li>
<li><strong>포인터 초기화:</strong> 선언과 동시에 <code>NULL</code>이나 유효한 주소로 초기화하세요.</li>
<li><strong>중괄호(<code>{}</code>) 생략 금지:</strong> if/for문 뒤에 코드가 한 줄이라도 중괄호를 쓰는 것이 안전합니다.</li>
<li><strong>우선순위는 괄호로:</strong> 연산자 우선순위가 헷갈릴 땐 무조건 <code>()</code>를 쓰세요.</li>
<li><strong>문자열 끝은 <code>\0</code>:</strong> C언어 문자열은 마지막 널 문자를 위한 1바이트가 더 필요함을 명심하세요.</li>
</ol>
