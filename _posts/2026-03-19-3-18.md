<h2><b>수요코딩후기(레디스 구현)</b></h2>
<p>레디스의 핵심기능 ,아키텍쳐 및 기술 스택, 핵심 설계 포인트, 품질 테스트 계획, 테스트 시나리오, QA TEST까지 문서로 작성했다.</p>
<p>명세서를 바탕으로 agent.md파일을 작성해서 codex로 바이브 코딩시 AI가 프로젝트 환경을 이해하고 올바르게 작업 할 수 있도록 했다.</p>
<p>임무 분담을 명확히 했던게 주요했다. Test 1인 , mongodb 와 우리가 구현한 레디스를 비교하는 시각화 그래프 그리는 1인, redis 구현 1인, mongodb 측 처리 1인 으로 진행했다.</p>

<p>내 담당은 테스트 코드 작성이였는데 계속 git main에 변경사항이 생기니 테스트를 언제 해야 할지 난감했다.</p>
<p>초기에는 직접 테스트를 돌려서 결과를 수동으로 공유했지만, codex로 main 브랜치 변동 시 테스트를 진행하고 결과를 노션에 반영하는 자동화를 진행했던게 주요했다.</p>

<h2><b>레디스</b></h2>
<p><b>remote dictionary server </b></p>
<p>메모리 기반의 고속 키-값 구조 데이터 저장소</p>
<p>캐싱, 세션 관리, 실시간 분석 및 메시지 큐등에서 활용되는 NOSQL 데이터베이스이다.</p>
<p>디스크 기반 DB보다 훨씬 빠른 읽기/쓰기 속도를 제공하여 웹 애플리케이션의 성능을 극대화하는 데 사용 된다.</p>

<p><b>핵심 특징</b></p>
<p>1. 인 메모리 성능 :  모든 데이터를 메모리에 상주시켜 초고속 조회 및 처리 속도 보장</p>
<p>2. 다양한 자료구조: List, Set, Sorted Set, Hash 등 다채로운 데이터 구조 지원</p>
<p>3. 영속성: 메모리 기반이지만 데이터를 디스크에 저장하여 서버 재시작 후에도 데이터 보존 가능</p>
<p>4. 고가용성 및 확장성: 레디스 클러스터를 통해 데이터를 샤딩하여 분산 저장하고, 시스템 확장이 용이함</p>

<p><b>주요 활용 사례</b></p>
<p>1. 캐싱: DB 부하를 줄이기 위해 자주 조회되는 데이터를 메모리에 캐싱하여 응답 속도 향상</p>
<p>2. 세션 관리: 웹 애플리케이션의 사용자 로그인 세션을 저장</p>
<p>3. 실시간 순위표:  Sorted Set을 활용한 게임 점수등 실시간 랭킹 기능</p>
<p>4. 실시간 채팅 및 알림: Pub/Sub 기능을 활용한 메시징</p>
