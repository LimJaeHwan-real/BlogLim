<h1>💻 2. 실습 코드</h1>
<pre><code>import asyncio
import time
import requests
import httpx

URLS = [
    "https://www.google.com",
    "https://www.python.org",
    "https://www.github.com",
    "https://www.naver.com",
    "https://www.wikipedia.org"
] * 2

# --- [방법 1] 동기 방식: 하나씩 순서대로 실행 ---
def fetch_sync():
    print("동기(Sync) 방식 시작...")
    start_time = time.perf_counter()
    results = []

    for url in URLS:
        response = requests.get(url)
        results.append(response.status_code)
        print(f"[Sync] {url} 완료")

    end_time = time.perf_counter()
    return end_time - start_time

# --- [방법 2] 비동기 방식: 대기 시간에 다른 작업 수행 ---

# 2-1. 개별 URL을 처리하는 코루틴
async def fetch_async_url(client, url):
    response = await client.get(url)
    print(f"[Async] {url} 완료")
    return response.status_code

# 2-2. 여러 비동기 작업을 조율하는 메인 코루틴
async def fetch_async():
    print("\\n비동기(Async) 방식 시작...")
    start_time = time.perf_counter()

    async with httpx.AsyncClient() as client:
        tasks = [fetch_async_url(client, url) for url in URLS]
        await asyncio.gather(*tasks)

    end_time = time.perf_counter()
    return end_time - start_time

# --- 프로그램 실행 시작점 ---
if __name__ == "__main__":
    sync_duration = fetch_sync()
    async_duration = asyncio.run(fetch_async())

    print("\\n" + "=" * 30)
    print(f"동기 방식 소요 시간  : {sync_duration:.2f}초")
    print(f"비동기 방식 소요 시간: {async_duration:.2f}초")
    print(f"비동기가 {sync_duration / async_duration:.1f}배 더 빠릅니다!")
    print("=" * 30)
</code></pre>
<h1>🧠 3. 비동기 vs 멀티스레딩</h1>
<h2>🔥 핵심 개념</h2>
<ul>
<li>비동기 = 한 명의 작업자가 여러 작업을 번갈아 처리하는 방식</li>
<li>멀티스레딩 = 여러 작업자가 동시에 작업을 처리하는 방식</li>
</ul>
<h2>📊 동시성 vs 병렬성</h2>
<table border="1" cellspacing="0" cellpadding="8">
<thead>
<tr>
<th>개념</th>
<th>설명</th>
</tr>
</thead>
<tbody>
<tr>
<td>동시성 (Concurrency)</td>
<td>하나의 스레드가 여러 작업을 번갈아 수행하여 동시에 진행되는 것처럼 보이게 하는 방식</td>
</tr>
<tr>
<td>병렬성 (Parallelism)</td>
<td>여러 스레드나 여러 코어가 실제로 동시에 작업을 실행하는 방식</td>
</tr>
</tbody>
</table>

<h2>❓ 왜 Python은 Async를 많이 사용할까?</h2>
<p>파이썬에는 <b>GIL(Global Interpreter Lock)</b>이라는 제약이 있습니다.<br />이 때문에 멀티스레드를 사용하더라도 CPU 바운드 작업에서는 기대만큼 성능 향상이 크지 않을 수 있습니다.</p>
<p>하지만 웹 요청, 데이터베이스 쿼리, 파일 읽기/쓰기 같은 <b>I/O 작업</b>은 대부분의 시간이 "응답을 기다리는 시간"입니다.<br />비동기는 바로 이 대기 시간을 다른 작업으로 채우기 때문에, 적은 리소스로도 많은 작업을 효율적으로 처리할 수 있습니다.</p>
<h1>🧾 4. 핵심 요약</h1>
<ul>
<li><code>async</code>는 여러 일을 동시에 &ldquo;보이게&rdquo; 처리하는 동시성 모델이다.</li>
<li><code>await</code>는 기다리는 동안 다른 작업에게 순서를 넘기는 핵심 지점이다.</li>
<li>파이썬 비동기는 특히 네트워크, API 호출, DB 통신 같은 I/O 작업에서 강력하다.</li>
<li>CPU 연산이 많은 작업은 비동기보다 멀티프로세싱이 더 적합한 경우가 많다.</li>
</ul>
